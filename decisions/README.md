# apnaPA Architectural Decision Records

ADRs record decisions that future work should understand before changing direction.

---

## When To Create An ADR

- Choosing between multiple valid architecture approaches.
- Establishing a pattern that future features should follow.
- Making a decision that is expensive to reverse.
- Replacing or superseding an earlier decision.

---

## ADR Template

```markdown
# ADR-001: Title

**Date**: YYYY-MM-DD  
**Status**: Proposed | Accepted | Superseded  
**Deciders**: Names or roles

## Context

What problem or constraint led to this decision?

## Options Considered

- Option A
- Option B

## Decision

What was chosen?

## Consequences

What becomes easier, harder, or constrained?

## References

- Related plan, issue, PR, or doc
```
